# AIViz Product Roadmap

**Vision:** 讓你的商品出現在 AI 搜尋結果裡。

**Business model:** 免費網頁診斷工具（吸引用戶）→ Shopify App（付費一鍵修復）

```
Phase A (現在)          Phase B (2-4月)         Phase C (4-6月)         Phase D (6-9月)
技術健檢                看得見的損失             競品比對+急迫感          Shopify App
─────────────────── → ─────────────────── → ─────────────────── → ───────────────────
讓用戶知道             讓用戶知道              讓用戶感到              讓用戶付費行動
「有問題」             「在虧錢」              「急迫」
```

## Phase B：看得見的損失（2-4 個月）

**目標：** 讓用戶知道自己「在虧錢」

**核心功能：**

### 1. AI 搜尋模擬（用 Gemini 模擬，不用真的呼叫 ChatGPT/Perplexity）

```
用戶的商品：XX 品牌無線耳機

模擬問題：「推薦我一款 2000 元以內的無線耳機」

AI 搜尋模擬結果：
  ❌ 你的商品：未被提及（原因：缺少評價資料、品牌辨識度不足）
  ✅ 競品 A：被推薦（原因：有完整評價 + 規格比較表）
  ✅ 競品 B：被推薦（原因：品牌知名度高 + 價格明確）

💡 修復建議：加入 aggregateRating + brand 資訊後，被推薦機率預估提升 40%
```

**技術實作：** 把商品資訊 + 假設的用戶問題餵給 Gemini，讓它模擬「作為 AI 搜尋引擎，你會推薦這個商品嗎？」

### 2. 關鍵字可見度

```
你的商品相關關鍵字：
  「無線耳機」     → AI 推薦你的可能性：低 ⬇️
  「藍牙降噪耳機」 → AI 推薦你的可能性：中 ➡️
  「通勤耳機推薦」 → AI 推薦你的可能性：無 ❌
```

**技術實作：** 從商品描述自動提取關鍵字，用 Gemini 模擬每個關鍵字的 AI 搜尋場景。

### 3. 地區設定（下拉選單）

中期開始需要地區，因為 AI 推薦結果因地區而異：
- 台灣 🇹🇼
- 日本 🇯🇵
- 美國 🇺🇸
- 其他

## Phase C：競品比對 + 趨勢追蹤（4-6 個月）

**目標：** 讓用戶感到「急迫」

**需要帳號系統（Supabase Auth + DB）**

### 1. 競品比對

```
你 vs 同類前 3 名

              你     競品A    競品B    競品C
AI 可見度     45%    78%     82%     61%
AI 搜尋提及   0 次   12 次   18 次    5 次
關鍵字覆蓋    2/10   8/10    7/10    5/10
```

### 2. 趨勢追蹤

```
你的 AI 可見度趨勢（需週期性掃描）：
  3月：32% → 4月：45% → 5月：45%（停滯中 ⚠️）
  競品 A：55% → 68% → 78%（持續上升 📈）
```

### 3. AI 品牌監測

即使平台（蝦皮、momo）擋了爬蟲，也能追蹤品牌在 AI 搜尋中的提及率：
- 定期用 Gemini 模擬「推薦 XX 類商品」
- 記錄品牌被提及的頻率和語境
- 這是動態渲染頁面的替代方案 — 不爬頁面，改追蹤 AI 搜尋結果

## Phase D：Shopify App（6-9 個月）

**目標：** 讓用戶付費行動

**功能：**
- 一鍵修復 structured data（自動注入 JSON-LD）
- 持續監測 + 自動優化
- AI 搜尋排名追蹤儀表板
- 關鍵字最佳化建議

**定價：**
| 方案 | 月費 | 內容 |
|------|------|------|
| Basic | NT$299/月 | 自動 structured data + 月度報告 |
| Pro | NT$799/月 | + AI 搜尋模擬 + 競品比對 + 關鍵字最佳化 |
| Enterprise | 客製 | + API 存取 + 多店面管理 |

**轉換路徑：**
```
用戶在免費工具看到報告
        ↓
「你有 8 個問題可以改善，修復後 AI 可見度預估提升 40%」
        ↓
「安裝 AIViz Shopify App → 一鍵修復所有問題」
        ↓
CTA 按鈕導向 Shopify App Store
```

## 決策記錄

| 決策 | 選擇 | 原因 |
|------|------|------|
| Repo 公開性 | 保持公開，Gemini prompt 放 env var | 利大於弊，等 Phase D 再考慮部分私有化 |
| 地區設定 | Phase A 不需要，Phase B 加入 | 技術健檢跟地區無關，AI 模擬才需要 |
| 非商品頁處理 | 偵測 + 引導貼正確 URL | 不需要用戶選「分析什麼」，讓他們貼正確的 URL |
| 動態渲染頁面 | 偵測 + 解釋原因 + 預告品牌監測 | 解釋不是工具壞了，同時製造中期功能的期待感 |
| AI 搜尋模擬 | Gemini 模擬（不真的跑 ChatGPT） | 成本低、可控、結果可重現 |
| 全方面報告 | Phase D 才用真實模擬器 | 真實模擬器成本高，留給付費用戶 |
